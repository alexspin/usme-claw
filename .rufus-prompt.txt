CRITICAL CONSTRAINTS from existing code — architect must verify all of these before designing:
- migrations/003_episodes.sql: existing episodes schema. New importance_score column added via migration 010.
- migrations/005_skills.sql: existing skills table. status field is 'candidate'|'active'|'retired', not a boolean.
- migrations/006_entities.sql: entity_relationships uses valid_from/valid_until for soft deletes — use this pattern, don't hard delete.
- src/schema/types.ts: all TypeScript interfaces for existing tables. New code must match or extend these, never contradict them.
- src/assemble/types.ts: RetrievalCandidate, InjectedMemory, AssembleResult shapes. Spreading activation adds items to the RetrievalCandidate pool before scoring — it does not bypass score() in score.ts.
- src/assemble/retrieve.ts: the TIER_QUERIES SQL map. Do not modify these queries. Spreading activation is a new function that runs after retrieve() returns.
- usme-openclaw/src/index.ts: InjectionLogEntry interface and writeInjectionLog(). Spreading fields must be added as optional fields to InjectionLogEntry — not a new log mechanism.
- usme-openclaw/src/index.ts: injection log path is /tmp/usme/injection.jsonl (configurable via USME_INJECTION_LOG env var). Dashboard reads from this path.
- nightly.ts: all LLM calls use tool_use + Zod safeParse. The new Haiku importance call must follow the same pattern already established in stepPromote/stepContradictions/stepSkillDraft.
- The Anthropic model alias for Sonnet is claude-sonnet-4-5 (matches existing usage in nightly.ts). Haiku is claude-haiku-4-5.
